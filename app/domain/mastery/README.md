# Current Mastery States

The repository approves the following app-facing states for the current Unit 1
loop, while final curriculum-level mastery labels and numeric cutoffs remain
unapproved:

- `insufficient_evidence`: not enough skill-linked evidence for a judgment
- `needs_review`: evidence exists, but automated confidence should stay low
- `developing`: documented error patterns suggest the learner still needs support
- `ready_for_next_step`: positive evidence is strong enough for conservative next-step planning

These are app-facing runtime states, not final curriculum policy.
