# Suji Format Selection Criteria

## Purpose
This document records a small, practical rule for choosing between `plain` and `webtoon` formats in Suji's learner-facing content.

It is based on two current Unit 1 examples:
- `S2` entry-point bridge in webtoon style
- `S4` verification-habit bridge in plain style

The goal is not to compare writing styles in general. The goal is to choose the format that helps Suji most at each learning moment.

## What We Learned From S2
- `S2` was mainly about helping Suji find the first starting point.
- Suji's likely problem there is freezing before the first move.
- In that case, webtoon style helps because:
  - the hesitation is visible right away
  - the first clue stands out more clearly
  - the emotional barrier feels smaller

Conclusion:
- when the learner is stuck before the first line, webtoon can make the entry point easier to see

## What We Learned From S4
- `S4` was mainly about checking whether an answer is really complete.
- Suji's likely problem there is unstable procedure, not failure to notice the task exists.
- In that case, plain style helps because:
  - the checking step is more direct
  - the procedure signal is easier to follow
  - extra scene elements do not compete with the math check

Conclusion:
- when the learner needs to stabilize a checking routine, plain is usually better

## Format Choice Criteria
- Choose `webtoon` when Suji is likely to freeze before starting.
- Choose `webtoon` when confidence is low and the first clue needs to feel lighter and easier to approach.
- Choose `plain` when the goal is to stabilize a procedure step.
- Choose `plain` when the task is verification, cleanup, comparison, or orderly checking.
- Choose `plain` when mixed concepts may already feel busy.

## Risks To Watch
- `webtoon` risk: the scene can become more noticeable than the math if the task is already procedurally heavy.
- `plain` risk: the content can feel too bare if Suji is anxious and cannot find the first action.

## Default Rule For Future Content
- Default to `plain`.
- Use `webtoon` selectively for entry-point moments, hesitation points, and confidence-sensitive starts.

This fits Suji's profile:
- weak starting point
- weak mixed-concept handling
- unstable procedure
- low application confidence

Use webtoon to open the door.
Use plain to steady the steps after that.
