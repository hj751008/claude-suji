# Suji Subskills: Prime Factorization

## Purpose
This document defines a short, practical source-of-truth breakdown for how Suji should learn Unit 1 `소인수분해`.

It is designed for Suji's current difficulty profile:
- weak starting point
- weak mixed-concept handling
- unstable procedure
- weak application confidence

This document does not define curriculum codes, numeric mastery thresholds, or scoring cutoffs.

## S1 Quick Diagnosis

### Stage Purpose
Check whether Suji is mixing up prime numbers, composite numbers, divisors, and factor-related vocabulary before starting procedure work.

### Small Subskills
- tell whether a number is prime, composite, or neither
- recall that `1` is not prime
- recognize that a prime number has exactly two divisors
- separate `소수`, `합성수`, `약수`, and `배수` language without mixing them

### Likely Stuck Points
- treating `1` as a prime number
- thinking all odd numbers are prime
- confusing "prime number" with "common factor" or "factor"
- answering from memory words instead of checking divisor structure

### Tutor Should Prompt First
Ask Suji to sort a few small numbers and explain one reason: "왜 그 수를 소수 또는 합성수라고 생각했어?"

### Early Success
- Suji correctly classifies a few small numbers including `1`
- Suji gives a divisor-based reason, not just a guess

## S2 Basic Entry

### Stage Purpose
Help Suji enter the unit by finding prime factors of one number without yet demanding fluent formal notation.

### Small Subskills
- find a divisor pair for a given number
- keep dividing until the remaining factors are all prime
- list all prime factors without stopping too early
- notice when a listed factor is still composite and must be broken again

### Likely Stuck Points
- stopping after finding one or two factors
- leaving a composite number in the result
- mixing up "factors of the number" and "prime factors only"
- losing track of what has already been split

### Tutor Should Prompt First
Ask: "지금 적은 수들 중에서 아직 소수가 아닌 것이 있을까?" before giving the next calculation step.

### Early Success
- Suji can break one number into prime factors completely
- Suji notices at least one incomplete result and fixes it

## S3 Expression Stabilization

### Stage Purpose
Stabilize how Suji writes prime factorization so the procedure becomes repeatable and less fragile.

### Small Subskills
- rewrite the final result using only prime numbers
- group repeated prime factors clearly
- move from split-work to final expression without dropping a factor
- recognize that different split orders can lead to the same final factorization

### Likely Stuck Points
- dropping a factor when rewriting
- copying intermediate work instead of the final factorized form
- thinking different division orders mean different answers
- writing a result that still contains a composite factor

### Tutor Should Prompt First
Ask: "마지막 줄에는 소수만 남았는지 먼저 같이 볼까?" and then have Suji rewrite once cleanly.

### Early Success
- Suji writes one complete prime factorization cleanly
- Suji accepts that two different split paths can match the same final result

## S4 Verification Habit

### Stage Purpose
Build a checking habit so Suji does not treat the first written answer as automatically correct.

### Small Subskills
- check whether every factor in the answer is prime
- check whether any prime factor is missing
- compare two candidate answers and decide which is complete
- multiply back or reason back to the original number when needed

### Likely Stuck Points
- accepting a partially correct answer
- checking only the product and not whether all factors are prime
- checking only primality and not whether anything is missing
- losing confidence when two answers look similar

### Tutor Should Prompt First
Ask Suji to check in this order: "소수만 있니?", "빠진 것은 없니?", "원래 수로 다시 맞니?"

### Early Success
- Suji can reject at least one incomplete or invalid answer with a reason
- Suji uses the same check order on her own once

## S5 Transfer Practice

### Stage Purpose
Help Suji use prime-factor information in nearby tasks without collapsing when concepts are mixed.

### Small Subskills
- use prime factorization to think about divisor count
- use prime factorization to decide what to multiply for a square number
- use factor structure when comparing common divisors or common multiples
- explain which part of the factorization matters for the new question

### Likely Stuck Points
- not knowing which earlier result to reuse
- freezing when the question is no longer labeled `소인수분해`
- mixing divisor-count ideas with GCD/LCM ideas
- needing reassurance before trying an application problem

### Tutor Should Prompt First
Ask: "이 문제에서 먼저 써먹을 소인수분해 결과가 뭐야?" before introducing a formula-like shortcut.

### Early Success
- Suji can start an application problem by writing or recalling a relevant factorization
- Suji solves one transfer problem with guided reasoning and can say why prime factorization helped
